class amqp_rpc_server.Server(amqp_dsn: str, exchange_name: str, executor: Callable[[bytes], bytes], content_validator: Optional[Callable[[bytes], bool]] = None, queue_name: Optional[str] = None, exchange_type: pika.exchange_type.ExchangeType = ExchangeType.fanout, max_reconnection_attempts: int = 5)#

Bases: object

Initialize a new RPC server with an underlying BasicConsumer handling the receiving and sending of messages

Parameters

  • amqp_dsn (str) – The Data Source Name pointing to the message broker. The message broker needs to support AMQPv0-9-1.

  • exchange_name (str) – The name of the exchange the server will use to receive new messages. If the exchange does not exist the underlying BasicConsumer will create the exchange

  • executor (Callable[[bytes], bytes]) – A method which handles the incoming message bytes and provides a response as bytes

  • content_validator (Callable[[bytes], bytes], optional) – A method which will validate the message content before it is passed to the executor

  • queue_name (str, optional) – The name of the queue which will be bound to the specified exchange, defaults to secrets.token_urlsafe()

  • exchange_type (pika.exchange_type.ExchangeType) – The type of exchange which will be used during the creation of the specified exchange. If the exchange already exists the exchange type supplied needs to match the one the exchange on the message broker has, defaults to pika.exchange_type.ExchangeType.fanout
